# iClassroom V2.0 Product Brief

本文是 V2.0 产品口径的简版说明，供团队、设计、AI 工具和评审人员快速理解项目。

---

## 1. 产品定位

iClassroom V2.0 是一个面向展厅、体验中心、培训和工作坊现场的轻量互动平台。

一句话：

```text
一个面向展厅运营的纯网页互动平台：管理者管账号，讲师用电脑运营，观众用手机扫码即入。
```

---

## 2. 目标场景

- 展厅讲解 / 导览中的现场答题、抢答与计分互动。
- 体验中心活动、培训、工作坊的现场互动与作品提交展示。
- 一场一房间、用完即走的临时性互动活动。

---

## 3. 角色模型

| 角色 | 是否有账号 | 设备 | 说明 |
| --- | --- | --- | --- |
| 管理者 | 有 | 电脑 | 管理讲师账号，后续查看运营概览 |
| 讲师 | 有 | 电脑 | 创建活动、发任务、评分、看板、导出 |
| 观众 | 无 | 手机优先 | 扫码入场、提交答案、查看结果 |

---

## 4. P0 闭环

```text
管理者配置讲师账号
→ 讲师登录建房
→ 生成房间号 / 二维码 / 链接
→ 观众扫码入场
→ 昵称验重并选择小组
→ 讲师发布任务
→ 观众提交文字 / 图片 / 文件
→ 讲师按小组审阅并打分
→ 小组排行榜更新
→ 大屏展示
→ 后台数据查看
→ 活动结束归档
→ 导出 Excel + 原件 Zip
```

---

## 5. 设计要求

- 管理者端和讲师端：电脑端优先，信息密度清晰，偏后台效率工具。
- 观众端：手机优先，短路径、低学习成本、适合现场扫码。
- 大屏端：适合现场全屏展示，突出排行榜、完成度、精选答案。
- 视觉：采用 AITIC 配色，阿里橙为主色，配深墨和暖中性。

---

## 6. 技术原则

- 纯网页，不做原生 App。
- 管理者 / 讲师必须登录。
- 观众无账号，活动级临时身份。
- 任务、提交、评分、排行榜需要实时同步。
- 数据持久保存。
- 图片 / 文件保留 30 天，保留期内支持导出原件。
- 生产环境必须使用固定主域名和 HTTPS。

---

## 7. 当前迁移提醒

旧版项目已经跑通了部分课堂 MVP 主流程，但 V2.0 新增了账号体系和管理者端。继续开发时不要沿用“讲师无登录 + teacherToken 管理房间”作为主架构。
